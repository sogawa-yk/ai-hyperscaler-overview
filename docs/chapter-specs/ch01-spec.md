# 第1章 アイデンティティ／認可 ― ワークフォース・CIAM・ワークロードID・委譲 ― 仕様書

## 章の概要

**章の目的**: アイデンティティ／認可の領域を「誰の・何の・どこへ伝えるID」という軸で切り、4社の製品をプロットして対訳・ギャップ・SWOTを示す。特に委譲・ID伝播とエージェントIDをケイパビリティ・カードで深掘りする。

**目標ページ数**: 24p

**所属する部**: 第II部 土台となる部品

## 前提知識

この章を読むために必要な前提:
- 序章の方法論（軸・原点・対訳・SWOT・分類手順）
- IAM・OAuth/OIDCの基礎概念

## 節の構成

### 1.1 軸の導入 ― アイデンティティを4つの問いで切る

**概要**: 「誰のID（ワークフォース／顧客＝CIAM）」「何のID（ワークロード／エージェント）」「どこのIdPを信頼するか（フェデレーション）」「呼び出しの連鎖で誰として動くか（委譲・ID伝播）」の4つの問いを軸として立てる。

**図表**:
- 図1.1: アイデンティティ領域を切る4つの軸

**キーポイント**:
- IDは「人/非人」と「単独/連鎖」で整理できる
- AIエージェント時代に「連鎖の中のID」が論点化する

### 1.2 4社プロット ― 各社のID製品を軸上に置く

**概要**: ワークフォース／CIAM／ワークロードID／フェデレーションの各軸に、AWS（IAM Identity Center, Cognito）／Azure（Entra ID, Entra External ID）／Google Cloud（Cloud Identity, Identity Platform）／OCI（IAM with Identity Domains）をプロットする。

**図表**:
- 表1.1: アイデンティティの4社プロット（軸×4社）

**キーポイント**:
- OCIはIdentity DomainsでワークフォースとCIAMを統合的に扱う
- ワークロードIDの実現方式は各社で差がある

### 1.3 対訳（他社→OCI）

**概要**: 代表製品のOCI相当を ≒／△／なし で示す。Cognito ≒ Identity Domains（CIAM）、Entra ID ≒ Identity Domains（ワークフォース）等。

**図表**:
- 表1.2: アイデンティティ対訳表（他社→OCI）

**キーポイント**:
- 多くは ≒ で対応するが、伝播・エージェント領域に △／なし が出る

### 1.4 ケイパビリティ・カード ― 委譲・ID伝播

**概要**: 委譲・ID伝播を定型カードで深掘りする。Entra OBO ↔ OCI Token Exchange（RFC 8693）+ Identity Propagation Trust（impersonationで代理元保持）の対応を示す。データ層への貫通は第5章へ予告する。

**図表**:
- 図1.2: 委譲・ID伝播のフロー（OBO と Token Exchange の対比）
- コード（リスト1.1）: RFC 8693 Token Exchange リクエスト例

**キーポイント**:
- 「代理元を保持したまま下流へ」が論点
- 標準（RFC 8693）に支えられている

### 1.5 ケイパビリティ・カード ― エージェントID

**概要**: AIエージェントに与えるIDを扱う。Entra Agent ID と、OCIでの組み立て方を対比する。エージェントIDは新興領域で陳腐化が速いことを確認日付きで明示する。

**図表**:
- 図1.3: エージェントIDの位置づけ（人・ワークロード・エージェント）

**キーポイント**:
- エージェントIDは発展途上
- OCIは既存のワークロードID＋伝播で組み立てる傾向

### 1.6 両方向ギャップとSWOTスライス

**概要**: 他社にありOCIにないもの（例: マネージドなエージェントID製品）、OCI固有（例: Identity Domainsの統合度）を整理し、SWOTでOCIの弱みを明示する。

**図表**:
- 表1.3: アイデンティティ領域の両方向ギャップとSWOTスライス

**キーポイント**:
- OCIの弱み（エージェントID製品の成熟度等）を省略しない

### 1.7 新顔の分類手順と確認日

**概要**: 未知のID系新製品を「誰の/何の/どこを信頼/連鎖でどう動く」の4問で分類する手順を示す。章末に確認日を記す。

**図表**:
- 図1.4: ID系新製品の分類フローチャート

**キーポイント**:
- 4つの問いに当てれば新製品も置ける

## 章末の理解度チェック問題

### 問題の方針
- 問題数: 4問
- 問題の種類: 概念の確認、判断問題、設計問題

### 問題案
1. ワークフォースID・CIAM・ワークロードIDの違いを述べよ（種類: 概念の確認）
2. Entra OBO はOCIの何に相当するか。記号付きで答えよ（種類: 判断問題）
3. ある新しい「エージェント認証サービス」を地図のどこに置くか、分類手順に沿って説明せよ（種類: 設計問題）
4. このID領域におけるOCIの弱みを一つ挙げよ（種類: 概念の確認）
