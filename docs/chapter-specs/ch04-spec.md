# 第4章 マネージドAI ― 基盤モデル・エージェント基盤・マネージドRAG・ガードレール ― 仕様書

## 章の概要

**章の目的**: 本書の重心の一つ。マネージドAIを「モデル提供層／RAG層／エージェント層／安全・運用層」という多層の軸で切り、4社（Bedrock / Foundry / Vertex / OCI Generative AI 系）をプロットする。MLプラットフォームとMCP対応も軸に含める。

**目標ページ数**: 30p

**所属する部**: 第III部 AIの中核

## 前提知識

この章を読むために必要な前提:
- 序章の方法論
- 第2章（AIを叩くホスト）、第3章（RAGの一次データ置き場）
- LLM・RAG・エージェントの基本概念

**概要**: Amazon Bedrock / Microsoft Foundry / Vertex AI / OCI Generative AI を基盤モデル提供の軸でプロットする。マルチモデル提供、ファインチューニング、リージョン/モデルの陳腐化を確認日付きで扱う。

**概要**: マネージドなRAG（Bedrock Knowledge Bases、Foundryの検索統合、Vertexの検索、OCIのRAG系）を並べ、構成要素（取り込み・チャンク・埋め込み・検索・生成）に分解して対訳する。
