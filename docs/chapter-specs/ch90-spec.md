# 統合章 地図の全体像 ― 対訳辞書・両方向ギャップ・AIワークロードSWOT ― 仕様書

## 章の概要

**章の目的**: 第1〜6章で領域ごとに描いた地図を1枚に統合する。全領域横断の(A)他社→OCI対訳辞書、(B)他社にありOCIにない一覧、(C)OCIにあり他社にない一覧、(D)AIワークロードSWOT、を集約・重複排除して提示する。必須コンテンツ5点の総まとめの章。

**目標ページ数**: 18p

**所属する部**: 第IV部 運用・統合・実践

## 前提知識

この章を読むために必要な前提:
- 第1〜6章すべて（本章は各領域章の集約）

## 節の構成

### 統.1 6領域を1枚に重ねる ― 全体地図の構造

**概要**: 6領域（ID・実行基盤・ストレージ・マネージドAI・データ側AI・o11y/ガバナンス）を1枚の地図に重ねる構造を示す。各領域章の軸を一望できるマップを提示する。

**図表**:
- 図統.1: AIワークロード全体地図（6領域×4社の俯瞰）

**キーポイント**:
- 領域ごとの軸が1枚で見渡せる
- ここまでの章の総索引として使える

### 統.2 対訳辞書（A）― 他社製品→OCI相当の総まとめ

**概要**: 各領域章の対訳を1つの辞書に集約する。代表製品を引くとOCI相当（≒／△／なし）が分かる横断テーブルを作る。重複を排除する。

**図表**:
- 表統.1: 他社→OCI 横断対訳辞書

**キーポイント**:
- 「他社製品を引けばOCI相当が分かる」一覧
- 本書の中核成果物の一つ

### 統.3 両方向ギャップ（B・C）― 双方向の差分一覧

**概要**: 「他社にありOCIにない」(B)と「OCIにあり他社にない」(C)を全領域から集約した二つの一覧を提示する。

**図表**:
- 表統.2: 他社にありOCIにない一覧（B）
- 表統.3: OCIにあり他社にない一覧（C）

**キーポイント**:
- 双方向で正直に並べる
- OCIの不足を隠さない

### 統.4 AIワークロードSWOT（D）― 4社の総合評価

**概要**: 各領域のSWOTスライスを統合し、4社それぞれのAIワークロードにおける総合SWOTを提示する。OCIの弱みを必ず含め、優位主張は出典のある範囲に限定する。

**図表**:
- 図統.2: 4社のAIワークロードSWOT総合
- 表統.4: SWOT統合表（領域横断）

**キーポイント**:
- 領域横断で強み/弱みのパターンが見える
- マーケ語を排した正直な総合評価

### 統.5 地図の使い方 ― 新顔を全体地図に置く統合手順

**概要**: 各領域章の「新顔の分類手順」を統合し、未知の新製品を全体地図のどこに置くかの統合フローを示す。読者が自走で地図を更新する方法をまとめる。

**図表**:
- 図統.3: 新製品を全体地図に置く統合分類フロー

**キーポイント**:
- まず領域を決め、次に軸上に置く
- 賞味期限のない能力の総仕上げ
