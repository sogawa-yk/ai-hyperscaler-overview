# 終章 公式RA読み比べ ― 同一題材で4社のリファレンスアーキテクチャを読む ― 仕様書

## 章の概要

**章の目的**: 本書のcapstone。同一題材（マネージドRAGアシスタント）について各社公式リファレンスアーキテクチャ（AWS SRA GenAI／Azure AI Landing Zones／Google Cloud Architecture Center／OCI Architecture Center）を並べ、全ボックスを本書の軸の語彙にマッピングする読解を行う。構築は一切しない。

**目標ページ数**: 16p

**所属する部**: 第IV部 運用・統合・実践

## 前提知識

この章を読むために必要な前提:
- 統合章（全体地図・対訳辞書・SWOT）
- 第1〜6章すべて（各領域の軸の語彙）

## 節の構成

### 終.1 capstoneの方法 ― 公式RAを軸の語彙で読む

**概要**: 本章のやり方を示す。同一題材を選び、各社の公式RAを取得し、図の各ボックスを本書の軸（ID・実行基盤・ストレージ・マネージドAI・データ側AI・o11y/ガバナンス）にマッピングする。構築せず読解のみであることを明記する。

**図表**:
- 図終.1: RA読解の方法（ボックス→軸へのマッピング手順）

**キーポイント**:
- 構築しない、読んで地図に載せる
- 同一題材だから4社を比較できる

### 終.2 題材の定義 ― マネージドRAGアシスタント

**概要**: 読み比べる共通題材（社内ドキュメントに対するマネージドRAGアシスタント）を定義する。必要な部品（認証・ホスト・データ置き場・モデル/RAG・データ層認可・監査）を本書の軸で列挙する。

**図表**:
- 図終.2: 共通題材の部品分解（本書の軸で表現）

**キーポイント**:
- 題材を軸の語彙で先に分解しておく
- これが4社RAの読み取りテンプレートになる

### 終.3 AWS SRA GenAI を読む

**概要**: AWSのRA（Security Reference Architecture / GenAIガイダンス等）を取得し、各ボックスを本書の軸にマッピングする。確認日付きで出典を明示する。

**図表**:
- 図終.3: AWS RA の軸マッピング

**キーポイント**:
- AWS固有の構成要素が軸のどこに載るか

### 終.4 Azure AI Landing Zones を読む

**概要**: AzureのAI Landing Zonesを同様にマッピングする。

**図表**:
- 図終.4: Azure RA の軸マッピング

**キーポイント**:
- ランディングゾーンの考え方を軸で読む

### 終.5 Google Cloud と OCI のRAを読む

**概要**: Google Cloud Architecture Center と OCI Architecture Center の該当RAをマッピングする。OCIを原点として他社との対応を確認する。

**図表**:
- 図終.5: Google Cloud RA の軸マッピング
- 図終.6: OCI RA の軸マッピング

**キーポイント**:
- OCI RAを原点に他社を相対配置
- 同一題材で4社の差が浮かぶ

### 終.6 4社RAの重ね合わせ ― 地図が読めることの確認

**概要**: 4社のマッピング結果を1枚に重ね、本書の軸の語彙で差分を語る。読者が「公式RAを地図に載せられる」状態に到達したことを確認し、本書を締める。

**図表**:
- 表終.1: 4社RAの軸別対応表（重ね合わせ）

**キーポイント**:
- 軸さえあればどの公式RAも読める
- 賞味期限のない能力の到達確認
